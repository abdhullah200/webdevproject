Team Page

Technical Description

The page is built using semantic and organized HTML and CSS techniques to maintain structure and usability

  • <nav>, <footer>, <div> - Used for semantic page layout.
  • <h1>, <p>, <ul>, <li> - Establishes a prominent hero area that introduces the site's mission
  • img tags with alt attributes are used for profile images, promoting accessibilty
  • <a> - Used as styled buttons linking to related content pages

The CSS implementation focuses on creating visual hierarchy, responsive design, and interactive elements:

  • Flexbox(display: flex, justify-content and align-items) creates a reponsive and centered layout for the team card section.
  • Each .mycard component uses bow-shadow for depth and border-radius for softer, modern appearences.
  • The hover effect (.mycard:hover) expands the card height and reveals hidden content using smooth transitions (transition: 0.4s ease-in-out, opacity and visibility)
  • Button styles use background-color, border-radius and hover stats to enhance user interaction.

These techniques ensure that the page is both functional and visually engaging while maintaining clean code structure.

Accessibility

Accessibility was thoughtfully considered in the development of the Team Page to ensure the content is usable by all visitors, including those using assistive technologies. All profile images include descriptive alt text, allowing screen readers to convey the visual content to visually impaired users.

Semantic HTML tags such as nav, footer, and appropriate headings were used to enhance the page's structure and navigability for screen readers. The color scheme was selected to maintain strong contrast specifically, a soft background color paired with dark text to ensure readability for users with visual impairments.

Additionally, the page includes large, clearly clickable buttons and uses a legible font (Poppins) with appropriate sizing to promote readability and ease of navigation across different devices.

Feedback Page

Technical Description

The page is built using semantic and organized HTML and CSS techniques to maintain structure and usability

  • Used a <form> with <fieldset> and <legend> to group sections.
  • Included text inputs, dropdowns, radio buttons, and a textarea for feedback collection.
  • Each input is labeled using <label> for clarity and accessibility.
  • A custom star rating system was created using <span> and JavaScript.
  • CSS styles include a soft background, centered layout, rounded corners, and clean spacing.
  • Form validation shows green or red borders based on input correctness.
  • Submit and Clear buttons have hover effects and are aligned using Flexbox.

These techniques ensure that the page is both functional and visually engaging while maintaining clean code structure.

Accessibility

Accessibility was carefully considered throughout the page. All form elements are paired with clear label tags, improving screen reader compatibility. Input fields and interactive components are keyboard-accessible.

High contrast was maintained between text and background, with dark text on a light background, aiding readability. Font sizing and spacing ensure comfort for users with visual impairments.

In addition, form feedback is visually indicated through color-coded borders and error messages, and semantic structuring (like using fieldset and legend) enhances assistive navigation.

Content Page

Technical Description

The page is built using semantic and organized HTML and CSS techniques to maintain structure and usability

  • <section> tags used for logical content blocks
  • <div> elements used for layout (cards, grids, hero)
  • <a> tag used for linking to external information (WFP site)
  • “Back to Top” button added using JavaScript for smooth scroll functionality
  • Clean, consistent design using the Poppins font
  • Cards for causes and efforts styled with hover effects
  • Hero section styled with a dark overlay to improve text readability
  • “Back to Top” button designed with CSS and enhanced using JavaScript

These techniques ensure that the page is both functional and visually engaging while maintaining clean code structure.

Accessibility

Accessibility was carefully considered in this content page to ensure all users can navigate and understand the information easily. All images include descriptive alt text, allowing screen readers to interpret visual content effectively. A clear heading structure using <h1>, <h2>, and <h3> tags improves document organization for assistive technologies.

The background color paired with dark text offers strong contrast, making content easier to read for visually impaired users. Interactive elements like links and buttons are large and easily clickable, and the layout remains user-friendly on both desktop and mobile devices, enhancing overall accessibility for everyone.

Challenges and Lessons Learned

During the implementation process, several challenges were encountered. For example, integrating interactive elements like the rating system needed JavaScript debugging to ensure smooth user experience. These challenges were resolved by iterative testing, seeking community resources, and methodical problem-solving. The key lessons learned include the importance of planning for accessibility from the start, writing clean code, and thorough testing on multiple browsers and devices.
