Home Page

Technical Description

The Home Page was structured using semantic HTML5 elements such as <nav>, <section>, <article>, and <footer> to provide clear structure and improve SEO and accessibility. The navigation bar (<nav>) includes links to other pages, enabling intuitive user navigation across the site. Content was grouped logically using <div> containers and headings like <h1> and <h2> to establish content hierarchy.

CSS was used to enhance visual appearance and layout through properties like background, padding, margin, and box-shadow. A responsive layout was achieved using Flexbox and CSS Grid, along with @media queries to ensure the page adapts to various screen sizes. Consistent color themes, font families, and hover effects were applied to maintain design uniformity and improve user interaction.

Accessibility

Accessibility was prioritized by including alt attributes on all images, providing descriptive text for screen readers. A clear heading structure was maintained to help assistive technologies interpret content order. The site uses sufficient color contrast between background and text to support users with visual impairments. All interactive elements like buttons and links are keyboard-navigable and include meaningful labels, ensuring accessibility for users with motor or visual difficulties.

Table Section

Technical Description

The table was created using semantic HTML elements including <table>, <thead>, <tbody>, <tr>, <th>, and <td> to ensure proper structure and readability of tabular data. These tags clearly define the header and body rows, making the content easy to scan and understand. CSS was applied to enhance the visual presentation by using properties like border for cell outlines, padding and text-align for spacing and alignment, and background-color to distinguish headers or alternate rows.

Additional styling includes :hover effects for interactivity and border-collapse: collapse; to give a cleaner look. Responsive design was supported using overflow-x: auto; on the table container to make the table scrollable on smaller screens, ensuring that content does not break layout or overflow.

Accessibility

Accessibility for the table was ensured by using the <caption> tag to provide a meaningful summary of the table's content. Each header cell uses the <th> tag to identify column headings, helping screen readers announce the correct context when navigating cells. Scope attributes like scope="col" or scope="row" were used where applicable to clarify relationships between cells. Sufficient contrast between text and background colors was maintained to improve readability for users with visual impairments.

Content Page

Technical Description

The Content Page was designed using semantic HTML5 elements such as <section>, <article>, <h2>, and <p> to clearly present the main information. These tags ensure the content is properly structured for both human readers and assistive technologies. The page layout is controlled through external CSS, using Flexbox for horizontal and vertical alignment and grid systems where multiple columns or cards were necessary.

Key CSS properties used include padding and margin for spacing, background-color to define content sections visually, and border-radius and box-shadow for aesthetic enhancements. Font styles were standardized using font-family and font-size to ensure readability. Responsive design was implemented using @media queries, so the page adapts to various screen sizes including tablets and mobiles.

Accessibility

Accessibility was addressed by applying alt text for images, ensuring that screen readers can describe visual content to users. Headings are organized hierarchically to allow easy navigation through assistive tools. Links include descriptive anchor text to help users understand their function without needing context. The color scheme maintains high contrast to aid visually impaired users, and all interactive content is operable using a keyboard.

Challenges and Lessons Learned

While developing the Home Page, one of the main challenges was achieving a responsive layout across different screen sizes. Initially, layout breakpoints were not working well, causing alignment issues. This was resolved by restructuring the layout using Flexbox and media queries more effectively. Consistent use of color schemes and navigation styling was also tricky at first, but creating a reusable style guide helped maintain visual consistency.


For the Content Page, the biggest challenge was organizing and presenting large sections of text without overwhelming the user. It required thoughtful structuring using semantic tags and proper heading levels. Additionally, balancing design with accessibility (like maintaining contrast and adding descriptive alt text) helped improve the clarity and usability of the page.


On the Table Page, the issue was making the table responsive. Tables naturally don’t fit well on smaller screens. The solution involved wrapping the table in a scrollable container using overflow-x: auto. Ensuring headers and cells were aligned and readable across devices was a key lesson in responsive design and layout troubleshooting.


Overall, these tasks taught valuable lessons on responsive design, accessibility, visual consistency, and user experience. Iterative testing and feedback played a major role in improving functionality and final presentation.

Compliance

All pages (Home, Content, and Table) follow the JANET Acceptable Use Policy. They are educational, non-commercial, and do not collect or track personal user data. Accessibility features such as alt text, semantic HTML, and keyboard navigation have been implemented to meet inclusive design principles.

The content is created in line with ethical and academic standards, with no sensitive or copyrighted material used. Styling and interactions are designed with user privacy and security in mind, ensuring a responsible and compliant web presence within an educational context.
