Splash Page

Technical Description

  • <meta http-equiv="refresh" content="4;url=Home.html"> is used to automatically redirect the user to the homepage after 4 seconds, giving enough time to present the splash animation.
  • The webpage uses the 'Poppins' font for a clean and modern aesthetic, typically imported via Google Fonts and applied globally through CSS.
  • Design aesthetics are enhanced with border-radius: 20px and box-shadow for smooth corners and depth.
  • The splash experience is animated using @keyframes:
    • bgZoom for background image scaling animation
    • fadeInUp for content entrance effect
    • bounce for animated loading dots
    • circleExpand for the expanding circle transition before redirect
  • The .mask-circle element creates a full-screen expanding circle transition just before redirection to enhance the visual flow and make the transition more fluid.

Accessibility

  • The lang="en" attribute on the HTML tag helps screen readers determine the correct language context.
  • The viewport meta tag allows the layout to adapt responsively on all devices, including mobile and tablets.
  • Responsive design is handled with @media queries to adjust text size, padding, and layout on smaller screens.
  • The main content is contained within the .card-container, which scales properly to different viewports for accessibility and usability.
  • Semantic elements and a logical heading structure (h2, h3) help assistive technologies interpret the page's structure.
  • Images, like the logo, use descriptive alt text to support users with visual impairments.

Volunteer Page

Technical Description

  • The page uses semantic HTML elements such as <section>, <form>, and <label> to maintain a clear content structure and enhance both usability and SEO.
  • User inputs are collected using a <form> element with fields like <input type="text"> and <input type="email">, accompanied by required attributes for basic validation.
  • <label> elements are linked to their corresponding input fields via the for attribute to improve accessibility and form clarity.
  • Styling is achieved using CSS with:
    • padding and margin to space out form elements
    • border-radius to round the corners of input boxes and buttons
    • box-shadow for subtle visual depth
    • hover and focus pseudo-classes to provide user feedback
  • A flexible layout is achieved using display: flex or display: grid to center the form and ensure responsiveness across different screen sizes.
  • The font 'Poppins' or a similar modern typeface is imported through Google Fonts for a clean and readable interface.

Accessibility

  • The lang="en" attribute on the HTML tag sets the document language for screen readers.
  • Each input field is clearly described using associated <label> tags to improve navigation for users relying on assistive technology.
  • Native HTML form controls ensure compatibility with keyboard navigation and screen readers.
  • Color contrast is designed to meet WCAG standards for readability, ensuring accessible visual design for users with low vision.
  • @media queries enable the form layout to adapt gracefully to various screen sizes, improving the mobile user experience.
  • Error messages or validation feedback can use aria-live regions to alert screen readers about changes dynamically.
