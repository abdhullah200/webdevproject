Splash Page


Technical Description


  • The tag <meta http-equiv="refresh" content="4;url=Home.html"> is used to automatically redirect users to the homepage after 4 seconds, allowing time to display the splash animation.

  • The page uses the 'Poppins' font, imported from Google Fonts, for a clean and modern look.

  • Design enhancements include border-radius: 20px for smooth corners and box-shadow to create visual depth.

  • The splash experience incorporates several animations via @keyframes:
    • bgZoom: background scaling effect
    • fadeInUp: smooth content entrance animation
    • bounce: animated loading dots
    • circleExpand: expanding circle transition before redirect

  • The .mask-circle element delivers a full-screen expanding circle to provide a seamless visual transition before redirection.

Accessibility


  • lang="en" helps screen readers correctly interpret the language.
  • The viewport meta tag ensures responsive rendering across devices.
  • Media queries adjust font sizes, spacing, and layout for smaller screens.
  • Main content is wrapped in .card-container to ensure proper scaling.
  • Semantic HTML and logical heading structure aid screen readers.
  • Images include descriptive alt text for accessibility.

Volunteer Page


Technical Description


  • The page uses semantic HTML elements such as <section>, <form>, and <label> to maintain a clear content structure and enhance both usability and SEO.

  • User inputs are collected using a <form> element with fields like <input type="text"> and <input type="email">, accompanied by required attributes for basic validation.
  • <label> elements are linked to their corresponding input fields via the for attribute to improve accessibility and form clarity.
  • Styling is achieved using CSS with:
    • padding and margin to space out form elements
    • border-radius to round the corners of input boxes and buttons
    • box-shadow for subtle visual depth
    • hover and focus pseudo-classes to provide user feedback
  • A flexible layout is achieved using display: flex or display: grid to center the form and ensure responsiveness across different screen sizes.
  • The font 'Poppins' or a similar modern typeface is imported through Google Fonts for a clean and readable interface.

Accessibility


  • The lang="en" attribute on the HTML tag sets the document language for screen readers.
  • Each input field is clearly described using associated <label> tags to improve navigation for users relying on assistive technology.
  • Native HTML form controls ensure compatibility with keyboard navigation and screen readers.
  • Color contrast is designed to meet WCAG standards for readability, ensuring accessible visual design for users with low vision.
  • @media queries enable the form layout to adapt gracefully to various screen sizes, improving the mobile user experience.
  • Error messages or validation feedback can use aria-live regions to alert screen readers about changes dynamically.

Content Page


Technical Description


  • This content page is structured using semantic HTML elements such as <nav>, <section>, and <h2> for clear content grouping and SEO benefits.
  • The fixed <nav> bar spans the top of the viewport and uses backdrop-filter and box-shadow for a frosted-glass look, enhancing aesthetic clarity while maintaining readability.
  • The hero section employs a full-screen background image with a semi-transparent dark overlay using the ::before pseudo-element to ensure that heading and paragraph text remain legible.
  • Major content sections such as "Hunger Situation", "Causes", "Efforts", "Solutions", and "References" are divided using <section> and <div> containers with appropriate class names for modular styling.
  • Cards in sections like .facts, .cause-list, and .card-list utilize box-shadow, border-radius, and transform: translateY() hover effects for visual interactivity and depth.
  • Responsive design is implemented using flex-wrap and a media query breakpoint at 1100px to ensure that help cards and content blocks adjust layout for smaller or larger screens.
  • The "Back to Top" button is styled to appear fixed in the lower-right corner, with hover effects and dynamic visibility (e.g., toggled via JavaScript).
  • The site uses the Google Font 'Poppins' across all elements for a consistent, modern visual tone.

Accessibility


  • Use of semantic sectioning elements such as <section> and heading tags (<h2>, <h3>) aids screen reader interpretation and improves navigation structure.
  • Interactive elements such as navigation links and buttons include clear hover/focus feedback to enhance usability for users navigating via keyboard.
  • Color contrast between text and background adheres to accessibility standards, especially for headings, links, and button hover states.
  • @media queries allow the layout to adapt for mobile users, ensuring readability and usability on smaller screens.
  • Clickable elements (like cards and icons) are adequately sized for touch accessibility and follow good spacing practices to prevent misclicks.

Challenges and Lessons Learned


A major hurdle was getting users comfortable with the feedback system. The first time it was put out, the response rate was low because it was just a basic input style and was too long. The fixes were less about the feedback process and more putting an interface together with engaging features like emoji ratings, where prompts were simplified and shortened and it worked.


Another issue came from maintaining visual consistency through the different aspects of the UI. There were small visual discrepancies between button styles, cards and hover effects that disrupted the user experience. But we were able to get there through reviewing our work, and standardizing the style.


Things to take into consideration are keeping user interactions simple, how visual feedback on engagements will contribute to user success, and consistency builds user trust. Finally, clear design systems and testing regularly helped to get to a final product.

Compliance


The web pages are in accordance with JANET regulations in that all pages are educational in nature, non-commercial and do not track or collect any personal information. The design is sensitive to the ethical and security concerns surrounding the publication of web pages and is consistent with the JANET Acceptable Use Policy from an educational perspective, in consideration of responsible web publication within a collegiate context.
