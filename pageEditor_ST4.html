User Profile Page


Technical Description


  • The user profile page uses semantic HTML elements such as <section> and <article> for clear content grouping and accessibility.

  • User information is displayed in cards styled using CSS Grid to ensure responsiveness and neat layout.

  • The page includes form controls for updating user details, with HTML5 validation attributes like required to ensure data integrity.

  • Profile picture upload is supported with descriptive alt attributes to enhance accessibility.

  • Styling and layout are consistent through external CSS files, following the project’s design system.

Accessibility


  • The lang="en" attribute enables screen readers to interpret the language correctly.
  • All form inputs are paired with <label> elements linked via for attributes for enhanced assistive technology support.
  • Keyboard navigation is fully supported, allowing users to tab through interactive elements.
  • Color contrast and font sizes are chosen to comply with WCAG 2.1 AA standards.
  • Images include meaningful alt text descriptions.

Sitemap Page


Technical Description


  • The sitemap page utilizes SVG graphics to visually represent the site’s navigation hierarchy with clickable interactive nodes.

  • Semantic HTML elements such as <nav> and <section> structure the content logically for SEO and accessibility.

  • SVG elements like <rect>, <line>, and <text> are used to create the site structure with links embedded within <a> tags.

  • Consistent CSS styling ensures the sitemap is visually coherent and responsive across devices.

Accessibility


  • The lang="en" attribute provides language context for screen readers.
  • Interactive SVG nodes are wrapped with semantic <a> links to support keyboard navigation and screen reader announcements.
  • All images and icons include descriptive alt attributes.
  • Color contrast complies with accessibility guidelines for legibility.
  • Logical tab order and visible focus states are provided for keyboard users.

Content Page


Technical Description


  • The content page uses semantic sectioning elements such as <section> and heading tags to organize topics clearly.

  • A fixed navigation bar uses backdrop-filter and box-shadow for a frosted glass effect while maintaining readability.

  • The hero section features a full-screen background image with a semi-transparent overlay using the ::before pseudo-element to ensure text contrast.

  • Content is divided into multiple sections including “Hunger Situation,” “Causes,” “Efforts,” “Solutions,” and “References” with consistent card styling and hover effects for user engagement.

  • Responsive design is implemented with flexbox and media queries for layout adjustment on smaller screens.

  • The page uses Google Fonts ('Poppins') for consistent typography across devices.

Accessibility


  • Use of semantic HTML5 elements improves screen reader navigation and SEO.
  • Navigation links and buttons include hover and focus styles for keyboard and assistive technology users.
  • Color contrast and font size comply with WCAG 2.1 AA for readability.
  • Responsive layout ensures content is readable and usable on all screen sizes.
  • Clickable elements have appropriate sizes for touch targets on mobile devices.

Challenges and Lessons Learned


Designing accessible and responsive pages across different content types posed challenges in balancing semantic HTML with complex styling requirements. Ensuring interactive elements were fully keyboard navigable and met accessibility standards required extensive testing.


Another challenge was managing consistent style and layout between pages while accommodating different content structures such as SVG-based sitemaps and user profile forms.


Lessons learned include the importance of semantic structure for accessibility, the value of external CSS for maintainability, and early validation testing to avoid issues in final deployment.

Compliance


All pages comply with JANET Acceptable Use Policy, ensuring content is educational and non-commercial, with no collection of personal data beyond what is necessary for user interaction. The design considers ethical publishing standards and maintains data privacy.
